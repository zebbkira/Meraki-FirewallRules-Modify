# Meraki L3 防火墙规则管理工具

一个面向 **Cisco Meraki MX 设备** L3 防火墙规则的批量管理 Web 工具。用网络工程师的实际运维场景设计：跨站点策略迁移、局部增量变更、Policy Object / VLAN 引用自动处理。把 Dashboard 上逐条手改的低效操作，变成可批量、可审计、可回滚的工程化流程。

## 为什么需要它

Meraki Dashboard 的 L3 防火墙编辑器是**逐条手工**操作，遇到下面这些常见场景非常低效：

- **新站点上线**：要把模板/参考站点的几百条防火墙规则搬过去，对象还要按目标组织重映射
- **批量变更**：同一段策略（比如某个 VLAN 的上网规则）要应用到几十个站点
- **局部增量**：只想往现有规则集里插入一小段策略，不想全量覆盖
- **对象引用**：规则里的 `OBJ(id)` / `GRP(id)` / `VLAN(tag)` 跨组织后 id 全变了，手工对不上

Meraki 的 API 只支持**全量覆盖**（PUT 整个规则列表），没有"追加一条""修改某条"的接口。这个工具在 API 之上做了工作流封装，解决上述所有问题。

## 核心能力

### 1. 全量导入（站点间策略迁移）
导出源站点的完整规则 → 编辑 → 导入到目标站点。自动处理：
- **Policy Object 重映射**：源组织的 `OBJ(506)` 按对象名/实际值在目标组织匹配对应对象，转成目标的 `OBJ(目标id)`
- **对象组（Group）处理**：组成员自动展开匹配
- **VLAN 引用**：映射成目标网络对应 VLAN 的实际子网（目标无该 VLAN 时进决策面板）

### 2. 增量合并导入（局部变更拼接）⭐
不全量覆盖，而是把一段策略**拼接**进目标站点现有规则：
- 在规则编辑表勾选一块规则 → 加入「增量暂存区」
- 选目标站点（支持批量多选）→ 预检 → 每个站点按 source 分块展示现有规则
- 指定插入位置（锚点）——补丁插在所选分块的 `deny any any` 之上
- **VLAN 标签透传**：`VLAN(1103).*` 保持原样（指向目标站点同号 VLAN）
- 多站点批量时，每个站点一个**标签页**，缺失对象统一决策一次

### 3. 批量多站点更新
选多个目标站点一次性预检 + 执行，每个站点独立备份、独立错误处理（单站点失败不阻塞其他）。

### 4. 对象索引查询
基于已缓存的组织对象索引，支持：
- **IP 查询**：输入 `10.1.1.1` → 找出所有包含该 IP 的 subnet（`10.0.0.0/8` 命中）和对象组
- **对象名模糊**：按名称片段匹配对象/组
- **FQDN 模糊**：匹配 fqdn 类型对象

### 5. 规则编辑体验
- **名称化显示**：表格里 `OBJ(506)` 直接显示对象名（如 `Corp_LAN`），底层仍存 id
- **对象名编辑**：输入对象名自动转成 `OBJ(id)`，支持内联搜索下拉
- **批量选中**：复制 / 删除 / 加入增量暂存
- **拖拽排序**：调整规则顺序（防火墙规则是顺序匹配的，位置很重要）

## 安装

### 前置要求
- Python 3.9+
- Meraki Dashboard API Key（**启用 API 权限**的组织）
- 网络可达 Meraki API 端点（中国区 `api.meraki.cn` / 国际区 `api.meraki.com`）

### 步骤

```bash
# 1. 克隆仓库
git clone https://github.com/zebbkira/Meraki-FirewallRules-Modify.git
cd Meraki-FirewallRules-Modify

# 2. 安装依赖
pip install -r requirements.txt

# 3. 配置环境变量
cp .env.example .env
#   编辑 .env 填入你的 API Key（见下方说明）

# 4. 启动
python app.py
```

浏览器打开 `http://127.0.0.1:5000` 即可使用。

### 获取 API Key

1. 登录 Meraki Dashboard
2. **Organization → Settings → API access**
3. 开启 API 访问，生成 API Key
4. 填入 `.env` 的 `MERAKI_API_KEY`

### 配置说明（`.env`）

```ini
# 必填：Meraki API Key
MERAKI_API_KEY=your_api_key_here

# 可选：监听端口（默认 5000）
FLASK_PORT=5000

# 可选：API 端点
#   中国区: https://api.meraki.cn/api/v1
#   国际区: https://api.meraki.com/api/v1
MERAKI_BASE_URL=https://api.meraki.cn/api/v1
```

> ⚠️ `.env` 含密钥，已在 `.gitignore` 中排除，**切勿提交到版本库**。

## 使用流程

### 场景 A：全量迁移（把 A 站点的策略整体搬到 B 站点）

1. 选择**源组织 + 源站点** → 「📥 导出规则」
2. 规则加载到编辑表，检查/微调
3. 点「📤 导入到新站点」→ 选**目标组织 + 目标站点**
4. 「🔍 预检」→ 系统匹配对象、找出缺失项
5. 缺失对象在决策面板逐条决定（创建对象 / 当字面值 / 跳过规则）
6. 「✅ 执行导入」→ 目标站点规则被**整体覆盖**（执行前自动备份）

### 场景 B：增量变更（把一段策略插进多个站点的现有规则）

1. 导出源站点 → 在规则表勾选要搬的那段规则
2. 点「📑 加入增量暂存」→ 暂存区显示补丁内容（VLAN/对象引用高亮）
3. 点「📤 导入到新站点」选目标组织 → 在暂存区添加**多个目标站点**
4. 「🔍 预检合并」→ 每个站点一个**标签页**，展示其现有规则按 source 分块
5. 每个标签页点选插入位置（补丁会插在该块的 `deny any any` 之上）
6. 「✅ 执行合并导入」→ 补丁拼接进各站点现有规则，原规则保留

### 场景 C：查对象（这个 IP 属于哪些对象/子网）

- 导出任意站点后（触发索引加载），在「🔎 对象查询」框输入：
  - IP（如 `10.1.1.1`）→ 命中所有包含它的 subnet 和组
  - 对象名片段（如 `Corp`）→ 模糊匹配
  - FQDN 片段（如 `google`）→ 匹配 fqdn 对象

## 安全与回滚

- **PUT 前校验**：任何非法 cidr / 裸 VLAN（目标 MX 不存在的）会在本地拦截，给出可读错误，不会让 API 返回难定位的 400
- **自动备份**：每次导入/合并前，目标站点的当前规则自动存到 `cache/backup_<network_id>_<时间戳>.json`
- **单站点隔离**：批量执行时某站点失败不影响其他站点，汇总报告逐站点显示结果
- **模板感知**：绑定模板的站点自动切换到模板 id 读写（修改会影响所有绑定站点，UI 会警告）

## 技术栈

- 后端：Python + Flask + Meraki Dashboard API v1
- 前端：原生 HTML/CSS/JS（单文件，无构建步骤）
- 缓存：对象索引入本地 JSON 文件（`cache/<org_id>_objects.json`），避免重复全量拉取

## 常见问题

**Q：导入时报"VLAN 引用需在该 MX 上存在"**
A：补丁或目标规则里有 `VLAN(tag).*`，但目标 MX 上没有这个 VLAN。检查目标站点是否配置了对应 VLAN，或在决策面板里手动指定子网。

**Q：导入报"site-to-site VPN spoke cannot be enabled with an invalid concentrator hub"**
A：这不是本工具的问题——目标模板的 VPN 配置本身有错误（某个 spoke 引用了无效 hub）。Meraki 会拒绝对该模板的任何写入。先去 Dashboard 修好 VPN 配置。

**Q：对象显示成 `OBJ(id)` 而不是名称**
A：对象索引导出后自动加载（控制台会打印加载数量）。跨组织时目标站点的对象 id 与源不同，名称可能解析不了——增量合并的锚点选择器已改为服务端按目标组织索引解析。

**Q：批量更新时漏选了某个站点的插入位置**
A：执行前会拦截并提示哪些站点未选锚点，自动切到该标签页引导你选择，不会静默插入。
